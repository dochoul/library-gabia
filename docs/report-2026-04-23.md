# 블로그 구성 보고서

작성일: 2026-04-23

## 개요

이 프로젝트는 Wisp CMS를 데이터 소스로 쓰는 Next.js 블로그입니다.
정적 마크업만 쌓는 방식이 아니라, CMS에서 글을 가져와 서버 렌더링으로 보여주고, 같은 콘텐츠를 기반으로 SEO 메타데이터와 OG 이미지까지 같이 생성하는 구조입니다.

핵심은 세 가지입니다.

- 콘텐츠는 Wisp CMS에서 가져온다.
- 화면은 Next.js App Router 기반으로 렌더링한다.
- 제목, 메타데이터, OG 이미지, RSS 같은 파생 산출물은 같은 설정값을 공유한다.

## 정보 구조

사이트 전체 설정은 [`src/config.ts`](/Users/david/Documents/work/library.gabia.com/src/config.ts)에 모여 있습니다.

여기서 관리하는 값은 다음과 같습니다.

- 블로그 ID
- 사이트 기본 URL
- 조직명
- 사이트 제목
- 사이트 설명
- 카테고리 목록

이 방식의 장점은 화면 여러 군데에서 같은 값을 따로 적지 않아도 된다는 점입니다.
홈, 상세 페이지, RSS, OG 이미지가 동일한 브랜드 값을 공유합니다.

## 콘텐츠 흐름

콘텐츠 조회는 [`src/lib/wisp.ts`](/Users/david/Documents/work/library.gabia.com/src/lib/wisp.ts)에서 생성한 Wisp 클라이언트를 통해 이뤄집니다.

구조는 단순합니다.

- `config.blogId`로 Wisp 클라이언트를 만든다.
- 홈에서 게시글 목록을 가져온다.
- 상세 페이지에서 단일 글을 가져온다.
- 카테고리와 태그 페이지도 같은 데이터 소스를 재사용한다.

즉, 데이터 소스는 하나이고 화면만 여러 개로 나뉜 구조입니다.

## 메타데이터와 검색 노출

검색과 공유에 필요한 값은 각 페이지가 따로 관리합니다.

- `title`
- `description`
- `openGraph.title`
- `openGraph.images`
- `icons`

특히 홈 페이지는 제목을 `Nabia 라이브러리`로 고정했고, `- Blog` 같은 부가 접미사는 제거했습니다.
이렇게 해서 실제 사이트 이름과 메타데이터가 어긋나지 않게 했습니다.

## 폰트 전략

이 프로젝트는 폰트를 두 층으로 나눴습니다.

- 일반 화면 폰트
- OG 이미지 폰트

### 일반 화면

전역 본문은 Pretendard를 씁니다.

- `globals.css`에서 Pretendard 외부 CSS를 불러온다.
- Tailwind의 `font-sans` 기본값도 Pretendard로 둔다.
- 기존 `next/font/google` 기반 IBM Plex Sans는 제거했다.

### OG 이미지

SNS 공유용 이미지는 서버가 직접 그립니다.
그래서 화면 CSS와 별개로 OG 이미지 생성 코드에서 폰트를 따로 로드합니다.

- Pretendard를 우선 시도한다.
- 원격 폰드 로드 실패 시 기존 IBM Plex Sans로 폴백한다.

이 방식은 화면과 이미지의 톤을 맞추면서도, 폰트 로드 실패에 대한 안전장치를 남겨 둔 구성입니다.

## 배포 관점

이 프로젝트는 로컬 개발값과 배포값이 분리될 수 있는 구조입니다.

- `NEXT_PUBLIC_BASE_URL`
- `NEXT_PUBLIC_BLOG_ID`
- `NEXT_PUBLIC_BLOG_ORGANIZATION`
- `NEXT_PUBLIC_BLOG_DESCRIPTION`

그래서 로컬 기본값보다 환경변수 우선순위를 명확히 보는 것이 중요합니다.
이 부분 때문에 제목이 잘못 노출되는 문제를 한 번 정리했습니다.

## 정리

현재 블로그는 다음과 같은 성격으로 구성되어 있습니다.

- Wisp CMS를 콘텐츠 원천으로 쓰는 Next.js 블로그
- 전역 설정값을 중심으로 화면, 메타데이터, OG 이미지가 함께 움직이는 구조
- Pretendard 기반의 한국어 친화적 타이포그래피
- SEO와 공유 미리보기를 함께 고려한 정적-서버 혼합 렌더링
